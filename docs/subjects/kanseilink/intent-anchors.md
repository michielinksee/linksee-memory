# KanseiLINK — Intent Anchors (declared directions)

> The **declared** strategy directions Drift OS reconciles against committed reality. Declared by
> curation (**declare-don't-mine** — never pattern-mined from logs). Each carries a `decision_mode`
> (the router → which detector + card behavior), a `lifecycle`, and a `confidence`. Superseded /
> retired anchors move to `decision-log.md`.

## Governing constraints
| ID | mode | conf | Constraint |
|----|------|:----:|-----------|
| **A4改** | constraint | 0.90 | Not full-stealth — split **public vs private**. PUBLIC = thesis · market education · docs · README · public-score concept · articles. PRIVATE = evaluation logic · scoring weights · proprietary framework · accumulated MCP intelligence · ranking/recommendation logic. |
| **A21** | constraint | 0.90 | MCP responses returning a **tool / score / recommendation / diagnosis** MUST include a canonical KanseiLINK app/deep-link **when applicable** — i.e. when returning a recommendation, returning a score, referencing an MCP profile, comparing options, or when the app has a next action. |

## Prohibitions (design principles)
- **Competitor avoidance** — do not chase Registry-only / security-scan / marketplace /
  enterprise-governance plays (avoid Moz-style focus-scatter).
- **Self-neutrality** — never rank own products preferentially.
